augment

augment ( obj , provider , ov )
Object augmentation takes static properties in the provider object (or an object literal containing augmentation members), and adds those as properties of the receiver object.
Parameters:
obj <Object> The receiver to be augmented
provider <Object> The provider object, whose properties will be supplied to the receiver
ov <Boolean> If true, properties supplied will be overriden on the receiver if already present.

augmentProto

augmentProto ( obj , provider , ov )
Object prototype augmentation takes prototype members in the provider object and adds those as prototype members of the receiver object.
Parameters:
obj <Object> The receiver to be augmented
provider <Object> The provider object, whose prototype members will be supplied to the receiver
ov <Boolean> If true, prototype members supplied will be overriden on the receiver if already present.

extend

extend ( sub , sup , proto , stat )
Simple object extension. Static members will not be inherited to the child class. The superclass and constructor properties are added to parent and child classes.
Parameters:
sub <Object> The subclass
sup <Object> The superclass from which to extend
proto <Object> prototype properties to add/override
stat <Object> static properties to add/override

mix

mix ( obj , ext , ov )
Augment an object by adding the static and prototype members of another object. By default, properties/methods are not overriden on the receiving object if they already exist. This is equivalent to augment + augmentProto.
Parameters:
obj <Object> The receiver which will accept the extension members
ext <Object> The provider object, whose static and prototype members will be supplied to the receiver.
ov <Boolean> If true, properties supplied will be overriden on the receiver if already present.
